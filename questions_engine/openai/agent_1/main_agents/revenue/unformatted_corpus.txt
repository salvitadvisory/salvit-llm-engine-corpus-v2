[DATE_FIELDS]

    - **Columns:**
      • column_name: The metric name (e.g., order_revenue).
      • description: A brief explanation of the metric.
      • rank: Priority (1 is highest).
      • weight: Influence on the overall score.
      • rationale: Explanation of its ranking.

[VARIABLE_RANKINGS]

  3. **Table 3: Date Fields Ranking**
    - **Columns:**
      • rank: Priority for date_field comparison.
      • date_field: The dynamic timeframe label.
      • weight: Its influence on the overall grade.
      • rationale: Explanation for the assigned weight.
    - **Purpose:** Rank which timeframes are most critical for analysis.

[DATE_FIELDS_RANKINGS]

  4. **Table 4: Data Partitions Rankings**
    - **Columns:**
      • rank: Priority.
      • partition: Grouping/category (e.g., New Subscription, Existing One-Off).
      • weight: Influence on overall performance scoring.
      • rationale: Explanation for ranking.
    - **Purpose:** Show how each business segment (partition) factors into the final revenue grade.

[DATA_PARTITIONS_RANKINGS]

  5. **Table 5: Variable Scoring**
    - **Columns:**
      • column_name: Metric name.
      • low_interval / high_interval: Numeric thresholds for grading.
      • grade: Letter grade corresponding to the interval.
      • rationale: Explanation of the thresholds.
    - **Purpose:** Map raw metric values to letter grades.

 [VARIABLE_SCORING]

  6. **Table 6: Explicit Time Periods**
    - **Columns:**
      • date_field: The dynamic label for the measurement period.
      • start_date: Start of the current period.
      • end_date: End of the current period.
      • start_date_comparison: Start of the comparison period.
      • end_date_comparison: End of the comparison period.
    - **Purpose:** Define the actual dynamic date ranges for current versus comparison data.

 [RESULTS_DATES]

  7. **Table 7: Insights on Analyzing Anomalies**
    - **Columns:**
      • finding: A concise statement of the observed anomaly.
      • Metrics_result_change: Details on how the metric changed (direction, magnitude).
      • plain_english_finding: A clear explanation in non-technical language.
      • possible_causes_to_evaluate: List of potential contributing factors.
      • possible_evaluation_and_key_diagnostic_checks: Suggested diagnostic steps.
      • possible_prescription: Recommended actions to address the anomaly.
      • possible_agent_to_call: Suggested roles or teams for follow-up.
      • rationale: Explanation for the recommendations.
    - **Purpose:** Tie metric changes to potential causes and actionable diagnostics.

[COMMON_CAUSES]


  8. **Table 8: Main Metrics Data**
    - **Columns:**
      • variable: The metric (e.g., order_revenue).
      • date_field: The dynamic timeframe label.
      • partition: Business grouping (e.g., “all” for entire business or specific segments).
      • current_value: Value for the current period.
      • comparison_value: Value for the comparison period.
      • change: Percentage change between periods.
      • math_score: A derived numeric score based on rankings.
      • weight: Combined weight for grading.
      • grade: Letter grade for the metric.
    - **Purpose:** Present core revenue performance data with explicit comparisons.

[TREE_DATA]

  9. **Table 9: Complementary Metrics Data** (Optional)
    - **Same columns as Table 8**
    - **Purpose:** Provide additional breakdowns of metrics when available; ignore if not present.

[COMPLEMENTARY_TREE_DATA]

    - **Definition:** A composite score (range 0–200) derived from the aggregated math_scores.

 [TREE_MATH_SCORE]
